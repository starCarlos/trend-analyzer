## 2. 状态定义

- `已完成`
  PRD 中定义的功能和验收标准，当前实现已基本满足
- `部分完成`
  主流程存在，但仍缺关键交互、约束或验收项
- `未完成`
  PRD 中要求的能力当前没有落地
- `不在本轮范围`
  已被 PRD 明确排除出 MVP

## 4. 逐项对比

### F1 搜索入口

PRD 要求：

- 支持 GitHub URL
- 支持 `owner/repo`
- 支持普通关键词
- 输入为空不可提交
- GitHub URL 自动规范化
- 最近搜索本地缓存 10 条

当前实现：

- GitHub URL、`owner/repo`、普通关键词都支持
- 前端空输入不会提交
- GitHub URL 会规范化为 `owner/repo`
- 浏览器本地会缓存最近搜索 10 条

判断：`已完成`

### F2 搜索结果页

PRD 要求：

- 趋势图
- 今日快照
- 相关内容列表
- 加入追踪按钮
- 支持 `7/30/90/all`
- 支持部分返回
- 必须展示数据可用性说明

当前实现：

- 趋势图、今日快照、内容列表、Track/Untrack 均已存在
- 时间范围支持 `7d/30d/90d/all`
- 后端搜索首包不等回填完成
- 前端有 availability 面板和回填 task 状态

判断：`已完成`

### F3 趋势图

PRD 要求：

- 只做分来源曲线
- GitHub 项目显示 star 日增历史
- 普通关键词显示 NewsNow 日快照累计曲线
- 无历史来源只显示当前点位，并提示“从今日开始积累”
- 至少同时支持 GitHub 和 NewsNow
- 图例可单独显隐
- 回填时已完成来源先渲染，未完成来源显示加载态

当前实现：

- GitHub star 历史已完成
- 普通关键词会把 NewsNow 日快照转换为本地累计曲线
- 前端能在回填完成前显示加载和 task 状态
- 支持多 source 展示
- 已有图例显隐交互
- 当普通关键词只有一个点位时，会明确提示“从今日开始积累”

判断：`已完成`

### F4 今日快照

PRD 要求：

- GitHub 当日 star 增量
- NewsNow 命中平台数
- NewsNow 命中条目数
- 最近更新时间
- 字段缺失显示 `N/A`
- 必须标明每个字段的数据来源

当前实现：

- 四个字段都已返回并展示
- 缺失值能显示 `N/A`
- 字段名本身已直接标明 GitHub / NewsNow 来源

判断：`已完成`

### F5 相关内容列表

PRD 要求：

- 按时间倒序
- 默认展示 20 条
- 支持按来源筛选
- URL 去重
- 数据源优先包含 NewsNow 和 GitHub 内容

当前实现：

- 已按时间排序
- 默认最多 20 条
- 内容去重已通过 `source + external_key` 或 URL 路径实现
- 前端与 API 都支持按 `all / newsnow / github` 筛选
- GitHub 项目类搜索已接入 GitHub Release / Issue / PR 内容抓取
- mock 与 real provider 都能返回 GitHub 内容项

判断：`已完成`

### F6 追踪列表

PRD 要求：

- 支持加入和移除
- 同一个规范化关键词不可重复创建
- 追踪状态在搜索结果页可见
- 加入追踪后，后续由定时采集持续补数

当前实现：

- 加入和移除已完成
- 相同规范化关键词不会重复创建
- 搜索页能看到 track 状态
- 后端已内置 scheduler，可按配置自动刷新 tracked 关键词

判断：`已完成`

### F7 冷启动与异步回填

PRD 要求：

- 首包先返回当前已知数据
- 后续异步回填 GitHub 历史
- 前端可轮询每个来源状态
- 某来源失败不影响其他来源保存和展示

当前实现：

- 搜索接口首包返回当前数据和 backfill job
- 回填任务有 `job + task` 两层状态
- 前端轮询 `/backfill-status`
- 后端按来源独立记录 success / failed / partial

判断：`已完成`
