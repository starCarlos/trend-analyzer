# TrendScope MVP PRD

> 版本：v1.0
> 日期：2026-03-17
> 状态：可执行草案

## 1. 文档目的

定义 TrendScope 第一阶段产品范围，确保产品、设计、后端、前端对 MVP 的目标一致。

本文只回答四个问题：

- 目标用户是谁
- MVP 解决什么问题
- MVP 明确做什么、不做什么
- 什么结果算上线可验收

## 2. 背景与问题

现有热度工具通常只能覆盖单一平台，或者只展示当前热榜，缺少两个关键能力：

- 把多个来源放到同一页面对照
- 对一个新词进行冷启动查询，而不是要求先追踪几天再看图

本产品的核心价值不是“做一个更大的热榜”，而是让用户在一次搜索里回答三个问题：

1. 这个词最近有没有升温
2. 热度主要来自哪个平台
3. 发生了什么内容事件

## 3. 目标用户

### 3.1 核心用户

- 关注开源项目热度的开发者
- 追踪 AI 工具和技术话题的内容创作者
- 需要快速判断“是否出圈”的产品经理或研究人员

### 3.2 首发用户画像

MVP 优先服务两类搜索对象：

- `GitHub 项目`
  例：`openclaw`、`anthropic/claude-code`
- `普通关键词`
  例：`AI Agent`、`MCP`

## 4. 产品目标

### 4.1 MVP 业务目标

- 用户输入一个词后，能在一个页面里看到趋势、当前快照和相关内容
- 对 GitHub 项目类搜索词，首次搜索即可看到历史趋势
- 对无法回溯历史的平台，系统明确告知“从当前时点开始积累”

### 4.2 成功指标

- 首次打开搜索结果页，后端在 `2s` 内返回首包
- 对 GitHub 项目类搜索词，历史回填任务在 `10s` 内完成 `90%` 请求
- 搜索结果页在部分数据源失败时仍可渲染
- 新用户无需预配置即可完成一次搜索和一次“加入追踪”

## 5. MVP 范围

### 5.1 必做功能

#### F1. 搜索入口

用户可输入：

- GitHub 仓库 URL
- `owner/repo`
- 普通关键词

验收标准：

- 输入为空时不可提交
- 输入 GitHub URL 时自动规范化为 `owner/repo`
- 最近搜索可在本地缓存 10 条

#### F2. 搜索结果页

页面固定包含四个模块：

- 趋势图
- 今日快照
- 相关内容列表
- 加入追踪按钮

验收标准：

- 页面支持 `7 天 / 30 天 / 90 天 / 全部` 切换
- 页面允许部分模块先返回，不能等待所有模块完成
- 页面必须展示数据可用性说明，例如“GitHub 历史已回填”“微博仅有今日快照”

#### F3. 趋势图

MVP 只展示分来源曲线，不做综合指数，不做自定义权重。

数据规则：

- GitHub 项目类：展示 GitHub star 日增历史
- 普通关键词：展示 NewsNow 来源的日快照累计曲线
- 无历史数据的来源：只显示当前点位，并带“从今日开始积累”的说明

验收标准：

- 至少支持 `GitHub` 和 `NewsNow` 两类来源同时展示
- 图例可单独显隐来源
- 冷启动过程中，历史回填完成的来源先渲染，未完成来源显示加载态

#### F4. 今日快照

展示当前查询词的当日摘要，不追求复杂评分。

MVP 字段：

- GitHub 当日 star 增量
- NewsNow 命中平台数
- NewsNow 命中条目数
- 最近一次更新时间

验收标准：

- 字段缺失时显示 `N/A`，不允许渲染报错
- 必须标明每个字段的数据来源

#### F5. 相关内容列表

按时间倒序展示与当前查询词相关的内容条目。

MVP 数据来源：

- NewsNow 热榜条目
- GitHub Release / Issue / PR 标题（仅 GitHub 项目类）
- 可选 RSS 补充，但不是上线阻塞项

验收标准：

- 默认展示 20 条
- 支持按来源筛选
- 同一条内容不可重复展示

#### F6. 追踪列表

用户可把当前搜索词加入追踪列表，后续由定时采集持续补数。

验收标准：

- 支持加入和移除
- 同一个规范化关键词不可重复创建
- 追踪状态在搜索结果页可见

#### F7. 冷启动与异步回填

首次搜索时，系统分两路工作：

- 快照路：先返回当前可拿到的数据
- 回填路：补 GitHub 历史数据，并把状态暴露给前端轮询

验收标准：

- 搜索接口首包必须包含当前已知数据和回填状态
- 前端轮询接口能看到每个来源的 `pending/running/success/failed`
- 任一来源失败时，其余来源结果仍可保存并展示

### 5.2 明确不做

以下功能全部移出 MVP：

- 综合指数
- 自定义权重和权重预设
- 热词发现页
- 事件标注
- 导出 CSV / PNG / SVG
- 飞书 / Telegram 推送
- Google Trends、Reddit、YouTube、Product Hunt 等多来源扩展

## 6. 关键用户流程

### 6.1 首次搜索 GitHub 项目

1. 用户输入 GitHub URL 或 `owner/repo`
2. 页面立即返回基础骨架和当前快照
3. 后端异步回填 GitHub star 历史
4. 趋势图逐步从“加载中”切到可视化曲线
5. 用户确认有价值后点击“加入追踪”

### 6.2 搜索普通关键词

1. 用户输入普通关键词
2. 后端拉取 NewsNow 实时命中条目
3. 页面显示当日快照和内容列表
4. 若本地已经积累过历史点位，则展示已有趋势
5. 若尚无历史，则明确提示“从当前开始积累”

## 7. 依赖与假设

- GitHub Token 可用，且具备基础额度
- NewsNow 采用公网 API 或自部署实例二选一
- 前后端都接受“部分数据成功”这一事实，不把全量成功作为前提
- MVP 默认单用户或轻量自用场景，不做复杂权限体系

## 8. 上线验收清单

- 可以从空库启动
- GitHub 项目首次搜索能完成冷启动并看到历史图
- 普通关键词首次搜索能看到 NewsNow 快照和内容列表
- 加入追踪后，定时任务能持续写入新点位
- 搜索、回填、采集失败都有可读错误状态
- Docker 启动文档可在新环境复现

## 9. 待决策事项

- 前端图表库：`ECharts` 或 `Chart.js`
- NewsNow 在 MVP 阶段使用公网实例还是直接自部署
- RSS 是否进入首发范围，还是完全后置
