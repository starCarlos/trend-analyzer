# README Hero Design Philosophy (2026-03-19)

## Name

Signal Table

## Philosophy

This README hero should feel like a product desk rather than a documentation screenshot dump. The composition needs to communicate that TrendScope is already real, already usable, and already opinionated about where attention should go. The center of gravity belongs to the trend line, while supporting views orbit it like instruments on the same surface.

The visual system should stay close to the product's own palette: warm paper, dark control surfaces, muted shadows, and amber signal accents. The point is not to repaint the app, but to frame it with enough restraint that the screenshots read as deliberate product evidence. Every crop should feel selected, not accidental.

Scale carries the message. The repository search view is the anchor because it expresses the app's strongest product promise in one glance: query, trend, signal summary, and content context. Keyword history and tracked workflow should support that promise from the side, showing breadth without competing for the first read.

The final artifact should look meticulously composed, as if it belongs at the top of a launch page. Spacing must feel measured, shadows soft but controlled, corners consistent, and every overlap intentional. Nothing decorative should distract from the fact that the underlying images are real screens from the current product.
